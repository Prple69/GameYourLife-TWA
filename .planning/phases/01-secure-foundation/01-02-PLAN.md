Add Telegram initData cryptographic signature validation to all backend endpoints. Replace the current spoofable `tg_id` query parameter pattern with verified identity extracted from the HMAC-SHA256 signed initData.

Purpose: Currently any client can pass any `tg_id` and the server trusts it blindly. After this plan, identity comes only from Telegram-signed data verified against the bot token. Spoofing becomes cryptographically impossible.

Output: `dependencies.py` with a FastAPI dependency that validates initData, `main.py` updated so every endpoint requires the dependency, and `api.js` updated to forward `initData` from Telegram SDK in a header on every request.

    Create `backend/app/dependencies.py` with the FastAPI dependency that validates Telegram initData:

    ```python
    from fastapi import Depends, HTTPException, Header
    from typing import Optional
    import json
    from telegram_init_data import validate, parse
    from app.config import get_settings

    async def verify_telegram_init_data(
        x_telegram_init_data: Optional[str] = Header(None, alias="X-Telegram-Init-Data"),
        settings=Depends(get_settings)
    ) -> dict:
        """
        FastAPI dependency: Validate Telegram initData HMAC-SHA256 signature.

        Reads initData from X-Telegram-Init-Data request header.
        Returns parsed init data dict on success.
        Raises HTTP 401 on missing, invalid, or expired initData.

        The returned dict contains:
          init_data["user"]["id"]  -> verified Telegram user ID (int)
          init_data["user"]["username"] -> Telegram username (str, may be absent)
          init_data["auth_date"]   -> Unix timestamp of signature (int)

        Library: telegram-init-data (pip)
        Validates HMAC-SHA256 and checks auth_date expiry (default 1 day).
        """
        if not x_telegram_init_data:
            raise HTTPException(
                status_code=401,
                detail="Missing Telegram initData header"
            )
        try:
            validate(x_telegram_init_data, settings.TELEGRAM_BOT_TOKEN)
            init_data_dict = parse(x_telegram_init_data)
            return init_data_dict
        except Exception:
            raise HTTPException(
                status_code=401,
                detail="Invalid or expired Telegram signature"
            )
    ```

    IMPORTANT: The `parse()` return value from `telegram-init-data` library returns a dict where `user` is a nested dict. Verify the actual API by checking the library: if `parse()` returns a string-keyed dict with "user" as a JSON string, wrap it: `init_data_dict["user"] = json.loads(init_data_dict["user"])`. Check the library docs/source to confirm the return type before finalizing.

    If `telegram-init-data` library is not found or has a different API, fall back to manual HMAC verification using Python stdlib:
    ```python
    import hmac
    import hashlib
    import urllib.parse
    from datetime import datetime, timezone

    def _verify_manual(init_data: str, bot_token: str) -> dict:
        parsed = dict(urllib.parse.parse_qsl(init_data, keep_blank_values=True))
        hash_value = parsed.pop("hash", None)
        if not hash_value:
            raise ValueError("No hash in initData")
        # Build data-check-string: sorted key=value pairs joined by \n, excluding hash
        data_check_string = "\n".join(
            f"{k}={v}" for k, v in sorted(parsed.items())
        )
        secret_key = hmac.new(b"WebAppData", bot_token.encode(), hashlib.sha256).digest()
        computed_hash = hmac.new(secret_key, data_check_string.encode(), hashlib.sha256).hexdigest()
        if not hmac.compare_digest(computed_hash, hash_value):
            raise ValueError("Signature mismatch")
        # Check expiry (1 day)
        auth_date = int(parsed.get("auth_date", 0))
        if (datetime.now(timezone.utc).timestamp() - auth_date) > 86400:
            raise ValueError("InitData expired")
        # Parse user field from JSON string
        if "user" in parsed:
            parsed["user"] = json.loads(parsed["user"])
        return parsed
    ```
    Use whichever approach works, but prefer the library.
