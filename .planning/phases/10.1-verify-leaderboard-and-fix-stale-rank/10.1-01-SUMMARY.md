## Decisions Made

- **Leaderboard.update placement** — placed **after** `db.commit()` AND `db.refresh(user)` so the User object reflects the post-mutation lvl/xp from DB (refresh re-reads). Putting it before commit would have ZADDed a transient pre-commit state; putting it after commit but before refresh would have used the in-memory mutation but missed any DB-side effects.
- **Two-test split** — plan asked for 1 test asserting ZADD; delivered 2 (one for ZADD success path with score equality, one for Redis-down tolerance). Independent regression coverage; minimal cost.
- **Defense-in-depth try/except** — `leaderboard.update` already has internal try/except; the router still wraps the call in another try/except so the per-call-site warning logs and execution-flow guarantee are explicit and don't depend on internal contract of the domain helper.
- **Minimal crud.add_reward removal** — kept the function body untouched aside from the leaderboard block. The function is dead code via the JWT migration but may have transient import-time effects elsewhere; risk-averse minimum diff.
